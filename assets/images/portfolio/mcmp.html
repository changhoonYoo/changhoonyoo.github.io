✉️ 메시징 플랫폼 개발

🚩 Description

프로젝트 소개 RabbitMQ를 활용한 대량의 메시지 처리 플랫폼 개발
개발 기간 2024.08 ~ 2024.10
직책 웹개발 팀장
담당 업무
  • 인증서버 연동 - 토큰 인증 요청 처리
  • 메시지 API 연동 - 비동기 처리, 과금 및 토큰 재처리
  • API 응답 처리 - 환불 및 DB 업데이트
  • VoIP 메시지 처리 - RabbitMQ로 Asterisk & FastAGI 연동
  • 무료 수신 거부 - 쉘스크립트로 인바운드 차단
운영 체제 Linux
개발 툴 Intellij, Postman, DBeaver, WinSCP, Mobaxterm, VSCode, JMeter, Figma
개발 스킬 Java, Spring Data JPA, JWT, Websocket, Spring Security, SQL, Apache, Nginx
데이터베이스 PostgreSQL, Oracle, Mysql, Redis

🚩 Details

Websocket을 활용한 처리 결과 전달 및 수신 메시지 알림

☑️ Stomp 프로토콜로 수신 문자 메시지에 대한 실시간 알림과 요청 메시지의 처리 결과를 전달합니다.

MSA 구조의 설계

☑️ 메시지 처리의 기능에 따라 프로세스를 분리하였습니다. 장애의 영향을 줄이고 개별 확장성을 높일 수 있게 설계하였습니다.

HTTP 클라이언트 라이브러리의 활용

☑️ HttpAsyncClient, FeignClient 등을 활용하여 상황에 맞게 동기/비동기적으로 처리하였습니다.

Apache, Nginx

☑️ 운영 웹서버는 Apache로 운영하였고, Stage 웹서버는 Nginx로 운영하였습니다.

토큰 만료 재처리 로직

☑️ 클라이언트 요청을 처리하는 도중에 토큰 만료를 우려하여 만료 시 재발급 후 재처리하는 로직으로 서비스 안정성을 향상시켰습니다.

Asterisk & FastAgi

☑️ 통화 연결 시 수신번호와 매핑되는 JSON 파일을 읽어 시나리오대로 처리되는 프로세스를 개발 담당했습니다.

쉘 스크립트를 활용한 무료수신거부

☑️ 인바운드 log 내의 수신 거부 요청 번호만을 수집하여 SCP를 통해 처리 담당 서버로 전달하는 스크립트를 작성하였고, crontab으로 설정하여 특정 시간마다 실행되도록 설계하였습니다.
